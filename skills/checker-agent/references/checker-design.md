# Checker Native MAF Design

## The "Distrust by Design" Philosophy
The Checker-Agent assumes the Maker has drifted until proven 
otherwise. It is the friction that ensures system stability.

### Check Protocol:
1. **Structural Check:** Does the output JSON match the schema?
2. **Logic Check:** Did the Maker actually do what the Planner asked?
3. **Audit Check:** Is there a permanent record of the action?
