# Runtime Directory (.pmcro/)

The `.pmcro/` directory is the Colony's authoritative runtime surface. It lives at `S:\.pmcro\` on the DevDrive. **Lifecycle docs in this directory are considered more authoritative than code when they conflict.**

---

## Directory Structure

```
S:\.pmcro\
├── identity.json              # Token source — {{company}}, {{author}}, {{project}}, etc.
├── config.json                # Runtime configuration (MaxLoops, LLM settings, etc.)
├── laws\                      # Colony law declarations (authoritative)
│   ├── EC-SYS-001.md
│   ├── EC-SYS-002.md
│   └── GTDDD-MANDATE.md
├── skills\
│   └── runtime-manifest.json  # Active skill manifest (generated by PmcroSkillLoader)
├── trails\                    # Execution audit trails (one per completed intent)
│   └── <ISO8601>.json
├── constraints\               # EarnedConstraints (written by Reflector on HALT/RETRY)
│   └── EC-<id>.json
└── generated\                 # Maker staging area for outputs before Checker validation
    └── <frame_id>\
```

---

## identity.json

The token resolution source for GTDDD-MANDATE compliance. All `{{token}}` placeholders in generated files are resolved from this file at write time by the Maker.

```json
{
  "company": "Tooensure LLC",
  "author": "Shawn",
  "project": "pmcro-agent-system",
  "namespace": "Tooensure.Pmcro",
  "repo_url": "https://github.com/tooensure/pmcro-agent-system",
  "devdrive_root": "S:\\",
  "skills_dir": "S:\\skills\\"
}
```

> **Never commit identity.json with real values.** The template version uses `{{token}}` placeholders resolved by `.template.config/template.json` at project generation time.

## trails/

Every completed intent produces a trail file at `.pmcro/trails/<ISO8601>.json`. Trails are immutable after writing — they are the audit log.

**Retention:** Trails are never auto-deleted. Archiving policy is manual.

**Querying trails:**

```powershell
# Last 10 trails
Get-ChildItem S:\.pmcro\trails\ | Sort-Object LastWriteTime -Descending | Select-Object -First 10

# All HALT dispositions
Get-Content S:\.pmcro\trails\*.json | ConvertFrom-Json | Where-Object { $_.final_disposition -eq 'HALT' }
```

## constraints/

EarnedConstraints are written by the Reflector immediately on hard conflict or HALT. The Planner must scan this directory before certifying any plan.

**Constraint lifecycle:**
1. Reflector writes `EC-<id>.json` on HALT
2. Planner scans on next cycle startup
3. Planner injects active constraints into `PlannerFrame.active_constraints`
4. Maker honors constraints during execution
5. Checker verifies constraints were honored
6. Constraint remains active until manually resolved or superseded

**Manual resolution:**

```powershell
# Mark constraint as resolved (rename, don't delete — for audit trail)
Rename-Item S:\.pmcro\constraints\EC-xxx.json EC-xxx.resolved.json
```

---

## generated/

Staging area for Maker outputs before Checker validation. If Checker fails, the staged output remains here (not promoted to final path). On COMPLETE, Maker promotes the file from `generated/<frame_id>/` to its final target path.

This ensures the target filesystem is never partially written — outputs are atomic.

---

## Authoritative Priority

When lifecycle docs in `.pmcro/laws/` conflict with code behavior, the **laws are authoritative**. Code must be updated to match the law declaration, not the reverse.
